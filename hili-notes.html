Challenges

Key Design Challenges & Solutions

01

Categorized Highlighting UX

Problem

Initial color-coding system required users to remember which color mapped to which category‚Äîcognitively taxing and unintuitive.

Solution

Switched to fixed category assignment. Users select category names directly (dropdown) instead of colors. App auto-assigns colors.

‚úì First-time users understood workflow in <30 seconds
02

Intuitive Workspace UX

Problem

Export buttons buried at bottom, no way to edit categories, organized highlights panel hidden below fold, no mobile right-click.

Solution

Reorganized with sticky bottom nav, "Activate Topic" button for appending highlights, editable category names, click-to-highlight menu on mobile.

‚úì User feedback improved from "confusing" to "intuitive"
03

PDF Loading & Persistence

Problem

Loading entire PDFs caused 5‚Äì10 second startup, high memory on mobile, lost highlights on refresh.

Solution

Lazy-loading pages on scroll, dual-layer persistence (localStorage + backend), Service Worker for offline sync.

‚úì Load times under 2 seconds, zero data loss
Outcomes

Key Learnings

1

Mobile-First Design Matters

Designing for desktop first and adapting to mobile later caused weeks of rework. Constraint drives innovation‚Äîdesign for mobile from Day 1.

2

Validate with Users Early

I spent 2 weeks optimizing the dashboard before getting feedback‚Äîonly to learn users didn't care about that part. Show testers a working version every 7 days.

3

Ship Core Feature Great

My biggest mistake was spending more time on dashboard than highlight workspace. The dashboard doesn't matter if the core feature sucks.
